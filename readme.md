# Team Finder

## О проекте

**Team Finder** — это веб-платформа для поиска единомышленников и формирования команд для совместной работы над IT-проектами. Сервис помогает разработчикам, дизайнерам, менеджерам и другим специалистам находить друг друга, объединяться в команды и воплощать идеи в жизнь.

### Основные возможности

- **Управление профилем**: регистрация, редактирование личной информации, загрузка аватара, добавление навыков.
- **Работа с проектами**: создание, редактирование, завершение проектов.
- **Поиск участников**: фильтрация пользователей по навыкам, просмотр профилей.
- **Участие в проектах**: возможность присоединяться к чужим проектам или приглашать других в свои.
- **Система навыков**: автодополнение при вводе навыков, создание новых при необходимости.
- **Безопасность**: валидация телефонных номеров и GitHub-ссылок, хэширование паролей, защита от несанкционированного доступа.

## Стек технологий

| Компонент                | Технология                                   |
|--------------------------|----------------------------------------------|
| Backend                  | Python 3.14, Django 5.2                      |
| База данных              | PostgreSQL                                   |
| Обработка изображений    | Pillow                                       |
| Валидация                | Django validators                            |
| Управление зависимостями | pip + requirements.txt                       |
| Конфигурация             | python-decouple (.env)                       |

## Требования к окружению

- Python 3.14 или выше
- PostgreSQL 15+ (опционально, для SQLite дополнительных требований нет)
- Git

## Установка и запуск

```bash
git clone https://github.com/WilSt78/team-finder-ad.git
cd team-finder

# Windows
python -m venv venv
source venv\Scripts\activate

# Linux / macOS
python3 -m venv venv
source venv/bin/activate

pip install -r requirements.txt
```

### Настройка переменных окружения

Создайте файл .env в корне проекта. Пример содержания:
env
| Переменная               | Назначение                                                                                                  |
|--------------------------|-------------------------------------------------------------------------------------------------------------|
| **DJANGO_SECRET_KEY**    | Секретный ключ Django. Можно сгенерировать через `get_random_secret_key` из `django.core.management.utils`. |
| **DJANGO_DEBUG**         | Режим отладки. Установите `True` во время разработки, `False` для продакшена.                               |
| **DJANGO_ALLOWED_HOSTS** | Список разрешённых хостов через запятую. Например: `localhost,127.0.0.1`.                                   |
| **POSTGRES_DB**          | Имя базы данных PostgreSQL.                                                                                 |
| **POSTGRES_USER**        | Имя пользователя PostgreSQL.                                                                                |
| **POSTGRES_PASSWORD**    | Пароль пользователя PostgreSQL.                                                                             |
| **POSTGRES_HOST**        | Адрес сервера БД. При локальной разработке — `localhost`.                                                   |
| **POSTGRES_PORT**        | Порт подключения к БД (по умолчанию `5432`).   

### Применение миграций
```bash
python manage.py migrate
```

### Создание суперпользователя (администратора)
```bash
python manage.py createsuperuser
```

### Запуск сервера
```bash
python manage.py runserver
```

После этого проект будет доступен по адресу: http://127.0.0.1:8000/

Админ-панель: http://127.0.0.1:8000/admin/
